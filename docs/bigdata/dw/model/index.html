数仓建模规范

# 阶段模型输出

阶段 模型 描述 产出
调研 业务模型 主要是对业务模型进行梳理，能否回答业务问题：用户推荐 渠道推广 用户画像 用户结构报表 运营活动报表 1.主要业务流程图最好由业务方提供，我们只梳理大概，如果太细节，一是花费时间，二是不一定准确 2.主要业务域划分基本上由业务系统决定，有多少个业务系统，就划分多少个业务域
领域模型 主要是对业务模型进行抽象处理，生成领域概念模型。用户宽表 订单宽表 商品宽表 1.数据域划分将业务域进一步抽象，划分出逻辑领域 2.事实表-维度表模型建设，覆盖核心逻辑 https://www.zhihu.com/question/25089273
设计 逻辑模型 主要是将领域模型的概念实体以及实体之间的关系进行数据库层次的逻辑化。 字段及指标的定义统一口径，约束及计算逻辑，理清逻辑依赖及数据依赖 1.指标分类梳理 2.数据分层设计 3.规范治理文档
实施 物理模型 主要解决，逻辑模型针对不同关系型数据库的物理化以及性能等一些具体的技术问题。 数据采集，存储，导出的策略

  • 分区策略: 指定分区字段，原则上每次调度只影响一个分区的数据
  • 调度策略: 指定作业及数据更新的频率，一般为每天/每小时
  • 保留策略: 分区数据存储时长，数据可恢复性，重要性考虑
  • 补跑策略: 任何情况需要恢复数据时，执行的调度方案

# 指标体系

指标类型 名称命名规则 名称举例 字段命名规则
原子指标 业务流修饰+基础度量词根 如支付(业务流修饰)金额(度量) order_amount
派生指标 日期修饰+业务限定+聚合修饰+原子指标 近七天(统计周期)使用红包订单(业务限定)累计(聚合方式)支付金额(原子指标) 7d_hongbao_sum_order_amount
衍生指标 日期修饰+业务限定+聚合修饰+原子指标+业务目标 近七天(统计周期)使用红包订单(业务限定)累计(聚合方式)支付金额(原子指标)占总支付金额比 7d_hongbao_sum_order_amount_rate

业务过程定义 + 表、字段 = 原子指标

原子指标 + 限定条件 = 派生指标

派生指标 + 逻辑运算 = 衍生指标
